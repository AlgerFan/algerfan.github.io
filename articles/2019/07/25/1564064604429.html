GitHub clone太慢？来加速你的GitHub clone

@AlgerFan  2019-07-25 0 评论 0 浏览
GitHub 

原因：git clone 特别慢是因为github.global.ssl.fastly.Net域名被限制了。只要找到这个域名对应的ip地址，然后在hosts文件中加上ip–>域名的映射，刷新DNS缓存就可以了

1. 查询域名global-ssl.fastly.Net和 github.com 公网地址

https://www.ipaddress.com/ 可以用这个查。

或者Dns检测|Dns查询 - 站长工具 查找检测列表里的TTL值最小的IP

分别查找github.global.ssl.fastly.net和github.com这两个网址的ip地址

2. 将ip地址添加到hosts文件

Linux：sudo gedit /etc/hosts
Windows：C:\Windows\System32\drivers\etc\hosts

保存退出

3. 刷新DNS缓存

修改完hosts还不会立即生效，你需要刷新DNS缓存，告诉电脑我的hosts文件已经修改了。

Linux：sudo /etc/init.d/networking restart

windows下刷新DNS的方法：

打开CMD

输入ipconfig /flushdns

重新clone就会发现速度飞起

如果不行就把github.global.ssl.fastly.net的配置删掉，只留github.com
