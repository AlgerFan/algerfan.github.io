（1）计算机网络概述——计算机网络复习笔记

@AlgerFan  2019-12-12 0 评论 0 浏览
计算机网络 

1.计算机网络在信息时代的作用

  • 数字化：计算机所处理的数据
  • 信息化：电话、QQ、购物等
  • 网络化：数字化和信息化都离不开网络，网络化分为三网：电信网、计算机网络、有线电视网络。计算机网络分为因特网、军网等其他网络。

互联网具有两个重要的基本特点：连通性和共享。

2. 因特网概述

计算机网络由若干结点和连接这些结点的链路组成。

  • 网络：计算机通过交换机连接起来
  • 互连网：网络与网络连接起来
  • 因特网：全球最大的一个互连网

internet（互连网）是一个通用名词，它泛指由多个计算机网络互连而成的计算机网络。

Internet（互联网）则是一个专用名词，它指当前全球最大的、开放的、由众多网络相互连接而成的特定互连网，它采用 TCP/IP 协议族作为通信的规则，其前身是美国的 ARPANET。

因特网发展的三个阶段

  • ARPANET 向互联网发展
  • 三级结构的因特网：主干网、地区网、校园网
  • 多层次的 ISP（运营商）结构的因特网

多层次的 ISP 结构的因特网

因特网的标准化由因特网协会 ISOC 制定，因特网体系结构研究委员会 IAB 由因特网研究部和因特网工程部组成。正式标准要经过三个阶段：互联网草案、建议标准、互联网标准。

3. 因特网的组成

互联网从工作方式上看，划分为两大类：

  • 边缘部分：用户直接使用的。
  • 核心部分：为边缘部分提供服务的。

3.1 主机之间的通信方式

  • 客户服务器方式（Client/Server 方式）
  • 对等方式（P2P）
    • 计算机既是客户机又是服务器，多用于快速下载

3.2 数据交换方式

  • 电路交换
    • 多用于打电话
    • 分为三个步骤：建立连接、通话、释放连接
    • 特点：适合于数据量较大的实时性传输，如：核心路由器之间
  • 分组交换
    • 用于计算机网络
    • 将数据分组直接发送，每段数据都有首部，即转发表，用于存储目标地址和源地址
    • 可进行互相转发，路由器也具有存储转发的功能
    • 优点：高效、灵活、迅速、可靠
    • 问题：时延、开销
  • 报文交换
    • 相比分组交换较长
    • 缺点：时延较长

三种交换方式的比较

4. 计算机网络的类别

计算机网络最简单的定义：一些互相连接的、自治的计算机的集合

5. 计算机网络的性能

  • 速率
    • 传输数据的速率
    • 单位为 b/s，Kb/s，Mb/s，Gb/s
  • 带宽
    • 数字信道所能传输的最高数据率
    • 单位为 b/s，Kb/s，Mb/s，Gb/s
  • 吞吐量
    • 单位时间内通过网络的数据量
    • 单位为 b/s，Kb/s，Mb/s，Gb/s
  • 时延：数据从网络的一段传送到另一端所需的时间
    • 发送时延：主机发送数据帧所需的时间。发送时延 = 数据块长度（bit）/信道带宽（bit/s）
    • 传播时延：电磁波在信道传播一定的距离需要的时间。传播时延 = 信道长度（m）/信号的传播速率（m/s）
    • 处理时延
    • 排队时延
  • 时延带宽积：传播时延*带宽
  • 往返时间：从发送方到收到接收方确认
  • 利用率
    • 信道利用率：有数据通过时间/（有 + 无）数据通过时间
    • 网络利用率：信道利用率加权平均值

6. 计算机网络的体系结构

体系结构分层带来哪些好处

  • 各层之间是独立的
  • 灵活性好
  • 结构上可分隔开
  • 易于实现和维护
  • 能促进标准化工作
    • 差错控制
    • 流量控制
    • 分段和重装
    • 复用和分用
    • 连接建立和释放

ISO 七层模型：国际标准化组织

  • 应用层：能够产生网络流量，能够和用户交互的应用程序
  • 表示层：开发人员进行加密、压缩
  • 会话层：服务和客户端建立的会话
  • 传输层：可靠传输建立会话、不可靠传输、流量控制
  • 网络层：IP 地址编址、选择最佳路径
  • 数据链路层：输入如何封装、添加物理层地址、Mac
  • 物理层：电压、接口标准

OSI/RM：互联网法律上的国际标准

TCP/IP：因特网事实上的国际标准

Network Protocols

网络体系结构：计算机网络各层及其协议的集合

网络排错：从底层到高层

6.3 开放系统信息交换的几个概念

  • 实体：交换信息的硬件或软件进程
  • 协议：控制两个对等实体通信的规则
  • 服务：下层向上层提供服务，上层需要使用下层提供的服务来实现本层的功能
  • 服务访问点：相邻两层实体间交换信息的地方
